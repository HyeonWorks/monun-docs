# 플러그인 개발

**이 문서는 제 플러그인들을 이용해 개발하는 방법에 관한 문서입니다**

제 플러그인에 원하는 모든 기능이 있을 수는 없습니다. 하지만, 플러그인을 처음부터 만들기가 어렵다면, 플러그인을 확장시킨 플러그인을 만들 수 있습니다. 즉, 제 플러그인의 API 기능들을 사용해 더 쉽게 원하는 기능을 구현하실 수 있습니다.

## 의존성 추가하기
API를 이용하시기 위해서는 플러그인에 대한 의존성이 필요합니다. 
1. 빌드된 플러그인을 프로젝트 안의 `/libs/` 디렉토리 안에 배치합니다. (디렉토리는 만들어야합니다)
2. 사용하는 빌드툴에 따라 아래 방법을 실행합니다.
### build.gradle
```
dependencies {
    compileOnly files("./libs/<PLUGIN>.jar")
}

```
### build.gradle.kts
```
dependencies {
    compileOnly(files("./libs/<PLUGIN>.jar"))
}
```

> <PLUGIN\>.jar 대신에 빌드된 플러그인의 파일명을 넣습니다.

3. 준비는 끝났습니다. 각 플러그인의 `개발하기` 문서를 확인해 API 사용법을 확인하세요.

## 흔한 실수
- 플러그인을 **절대** `shade`하지 마세요. 버킷은 이를 플러그인으로 생각하고 중복 플러그인 오류를 냅니다.
- 플러그인은 따로 플러그인 디렉토리에 배치해야합니다.
